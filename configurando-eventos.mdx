## Visão geral do funcionamento

De forma simples, o fluxo funciona assim:

1. o Cardápio Digital carrega os dados do estabelecimento;

2. se houver um **Google Tag Manager válido** cadastrado, o container é inicializado;

3. conforme o cliente navega no cardápio, abre produtos, adiciona itens, escolhe entrega, aplica cupom e finaliza o pedido, o sistema envia eventos para o `dataLayer`;

4. o Google Tag Manager lê esses eventos;

5. suas tags configuradas no GTM podem então enviar esses dados para o Google Analytics 4 e outras ferramentas compatíveis.

### Em termos técnicos

O Cardápio Digital utiliza o `dataLayer` do Google Tag Manager para empurrar eventos estruturados de navegação e conversão. Esses eventos podem conter, conforme o caso:

- nome do evento;

- dados de ecommerce;

- tipo de pedido;

- forma de pagamento;

- cupom aplicado;

- identificador da transação;

- dados complementares do usuário em eventos de conversão, quando disponíveis.

Isso permite uma estrutura de rastreamento mais confiável, organizada e compatível com o fluxo moderno de ecommerce no GA4.

## O que o Cardápio Digital entrega para o GTM

A BeeFood cobre o funil principal de navegação e compra do Cardápio Digital, desde a entrada no menu até a conclusão do pedido.

Os eventos enviados permitem acompanhar:

- visualização do cardápio;

- seleção de unidade;

- visualização de listas de produtos;

- visualização de produto;

- visualização e seleção de promoções;

- adição e remoção de itens do carrinho;

- abertura do carrinho;

- início do checkout;

- visualização das etapas do checkout;

- seleção do tipo de pedido;

- envio de informações de entrega;

- seleção da forma de pagamento;

- aplicação de cupom;

- compra concluída;

- falha no pagamento.

## Fluxo completo da jornada do cliente

## 1. Entrada no Cardápio Digital

Quando o cliente acessa o Cardápio Digital e o carregamento é concluído, a BeeFood pode enviar eventos relacionados ao início da navegação.

### Eventos dessa etapa

#### `menu_view`

Indica que o cliente visualizou o cardápio.

**Uso prático:**\
Pode ser usado para medir acessos reais ao menu e criar públicos de visitantes.

#### `restaurant_unit`

Indica qual unidade ou restaurante foi selecionado no contexto daquele cardápio.

**Uso prático:**\
É útil para operações com mais de uma unidade ou para relatórios segmentados por loja.

#### `menu_name`

Indica qual **cardápio/experiência de menu** está sendo visualizado naquele momento no Cardápio Digital.

**Uso prático:**\
Ajuda a diferenciar experiências, menus ou contextos diferentes quando houver mais de uma estrutura de cardápio ativa.

Abaixo, um exemplo simples de `menu_view`. Nesse caso, o evento mostra que o cardápio foi carregado e já informa a unidade e o nome do menu utilizado no contexto da navegação:

```json
{
  "event": "menu_view",
  "restaurant_unit": "BeeFood - Cardápio Digital",
  "menu_name": "teste"
}
```

> **Observação:** no fluxo atual, `restaurant_unit` pode aparecer como evento complementar dessa entrada no cardápio, mas neste artigo ainda não há um JSON separado capturado para ele. Vale incluir depois, se você quiser documentar visualmente todos os eventos de forma individual.

## 3. Interação com o carrinho

Quando o cliente começa a montar o pedido, entram os eventos ligados ao carrinho. Essa etapa já sinaliza uma intenção de compra mais concreta, porque o cliente deixa de apenas navegar e passa a montar um pedido.

### Eventos dessa etapa

#### `add_to_cart`

É disparado quando um item é adicionado ao carrinho.

**Normalmente pode incluir:**

- produto;

- valor;

- quantidade;

- dados de ecommerce.

**Exemplos de uso:**

- medir quais produtos geram mais intenção de compra;

- comparar produtos visualizados com produtos adicionados;

- avaliar se promoções, combos e destaques estão convertendo em ação.

No exemplo abaixo, o evento `add_to_cart` mostra que um item foi incluído no carrinho, junto com os dados principais de ecommerce, como valor, moeda e item adicionado:

```json
{
  "event": "add_to_cart",
  "restaurant_unit": "BeeFood - Cardápio Digital",
  "menu_name": "teste",
  "ecommerce": {
    "currency": "BRL",
    "value": 20,
    "items": [
      {
        "item_id": "2024769",
        "item_name": "Combo One Cheddar",
        "item_category": "Geral",
        "price": 20,
        "quantity": 1
      }
    ]
  }
}
```

#### `remove_from_cart`

É disparado quando um item é removido do carrinho.

**Exemplos de uso:**

- identificar desistência de itens específicos;

- entender quais produtos têm maior taxa de remoção;

- comparar adição e remoção de itens ao longo da jornada.

## 4. Início do checkout

Quando o cliente deixa a etapa inicial do carrinho e segue no processo de pedido, a BeeFood envia eventos ligados ao checkout.

Essa parte do funil é uma das mais importantes, porque mostra onde o cliente realmente começa o processo de conversão e em qual etapa ele continua ou abandona.

- comparar carrinhos abertos com checkouts iniciados;

- usar como base para públicos de remarketing e análise de abandono.

## 7. Conversão ou falha

No fechamento do pedido, o Cardápio Digital informa ao GTM se a compra foi concluída com sucesso ou se houve falha.

Essa é a etapa mais sensível do funil, porque separa intenção de resultado real.

### Eventos dessa etapa

#### `purchase`

É o evento de compra concluída.

Este é o principal evento de conversão do funil.

Ele pode incluir informações como:

- identificador da transação;

- valor da compra;

- itens do pedido;

- frete;

- desconto;

- cupom aplicado.

**Uso prático:**\
É o evento ideal para configurar conversões no Google Analytics 4 e acompanhar vendas reais.

Abaixo, um exemplo corrigido de `purchase`, representando uma compra concluída com sucesso:

```json
{
  "event": "purchase",
  "restaurant_unit": "BeeFood - Cardápio Digital",
  "menu_name": "teste",
  "ecommerce": {
    "transaction_id": "47997916",
    "currency": "BRL",
    "value": 31.5,
    "items": [
      {
        "item_id": "2024769",
        "item_name": "Combo One Cheddar",
        "item_category": "Geral",
        "price": 20,
        "quantity": 1
      }
    ],
    "coupon": "DESCONTO10"
  },
  "coupon": "DESCONTO10",
  "discount_value": 10,
  "order_type": "pickup",
  "user_data": {
    "first_name": "Roberto Barretos",
    "email": "roberto.barretos26@gmail.com",
    "address": {
      "street": "Rua Luiz Góes, 123, Conceiçãozinha",
      "city": "Guarujá",
      "region": "SP",
      "postal_code": "11472040",
      "country": "BR"
    }
  },
  "checkout_step": "confirmation",
  "shipping_tier": "pickup",
  "payment_type": "debit_card"
}
```

> **Aviso importante:** no exemplo anterior que você tinha, o evento `purchase` estava com `failure_reason`. Isso não faz sentido para compra concluída. Esse campo deve ficar apenas no evento `payment_failed`.

#### `payment_failed`

É disparado quando ocorre falha no pagamento ou quando o pedido não é concluído com sucesso.

**Uso prático:**

- identificar problemas na etapa final;

- monitorar falhas por meio de pagamento;

## Exemplo prático de configuração no GTM

Depois de entender os eventos que o Cardápio Digital envia, o próximo passo é transformar isso em rastreamento utilizável dentro do Google Tag Manager. A forma mais prática de começar é usando o evento `purchase`, porque ele representa a conversão final do pedido.

A lógica do processo dentro do GTM costuma seguir esta ordem:

1. criar um **acionador** para escutar o evento;

2. criar as **variáveis da camada de dados** para capturar os valores necessários;

3. criar a **tag de evento do GA4** para enviar essas informações ao Google Analytics.

### Exemplo: criar uma tag para compra concluída

#### Etapa 1 — Criar o acionador do evento `purchase`

No Google Tag Manager:

1. acesse o container desejado;

2. no menu lateral, clique em **Acionadores**;

3. clique no botão **Novo**;

4. clique em **Configuração do acionador**;

5. escolha a opção **Evento Personalizado**;

6. no campo **Nome do evento**, informe `purchase`;

7. mantenha o acionamento para esse evento;

8. salve.

Esse acionador será responsável por escutar o momento em que o Cardápio Digital enviar o evento `purchase` ao `dataLayer`.

<Image src="https://blob-cdn.documentation.ai/org-daf9910e-bdcf-4cb1-ab83-4cd8623e3f5e/doc-1c1fae12-7678-4e2e-90b6-bf66ffac55be/1773961327960-ck7xlxv9o7f-criacao-acionador-purchase.criacao-acionador-purchase" width="1900" height="952" alt="" caption="Configuração de um acionador do tipo Evento Personalizado no Google Tag Manager, com o nome do evento purchase, responsável por escutar o momento em que a compra é concluída no Cardápio Digital." />

#### Etapa 2 — Criar as variáveis da camada de dados

Com o acionador pronto, o passo seguinte é capturar os dados que vierem dentro do evento. Para isso, você deve criar variáveis da camada de dados.

No Google Tag Manager:

1. no menu lateral, clique em **Variáveis**;

2. na seção de variáveis definidas pelo usuário, clique em **Nova**;

3. clique em **Configuração da variável**;

4. escolha **Variável da Camada de Dados**;

5. no campo **Nome da variável da camada de dados**, informe o valor desejado.

Os primeiros exemplos mais úteis para esse evento são:

- `ecommerce.transaction_id`

- `ecommerce.value`

- `ecommerce.currency`

- `ecommerce.coupon`

Se quiser começar com apenas uma para fins didáticos, a melhor escolha é `ecommerce.transaction_id`, porque ela representa diretamente o identificador da compra.

<Image src="https://beetech-imagens.s3.sa-east-1.amazonaws.com/Screenshot_2026-03-19_at_20.06.14.png" width="1901" height="951" alt="" caption="Configuração de uma Variável da Camada de Dados no Google Tag Manager, usando ecommerce.transaction_id para capturar o identificador único da compra enviada pelo Cardápio Digital." />

#### Etapa 3 — Criar a tag de evento do GA4

Depois de preparar o acionador e as variáveis, você pode criar a tag que enviará o evento ao Google Analytics 4.

No Google Tag Manager:

1. no menu lateral, clique em **Tags**;

2. clique em **Nova**;

3. clique em **Configuração da tag**;

4. escolha **Google Analytics: evento do GA4**;

5. selecione a sua tag de configuração do GA4;

6. no campo **Nome do evento**, informe `purchase`;

7. adicione os parâmetros que quiser enviar, utilizando as variáveis criadas anteriormente;

8. associe ao acionador criado para `purchase`;

9. salve.

Uma configuração inicial bastante útil pode incluir:

- `transaction_id`

- `value`

- `currency`

- `coupon`

Assim, quando o evento `purchase` ocorrer no Cardápio Digital, o GTM enviará essas informações para o GA4.

<Image src="https://blob-cdn.documentation.ai/org-daf9910e-bdcf-4cb1-ab83-4cd8623e3f5e/doc-1c1fae12-7678-4e2e-90b6-bf66ffac55be/1773961910673-6p4g558ly94-criacao-tag.criacao-tag" width="1900" height="950" alt="" />

#### Etapa 4 — Testar antes de publicar

Antes de publicar qualquer alteração, o ideal é validar tudo no modo de pré-visualização.

No Google Tag Manager:

1. clique em **Visualizar**;

2. abra o Cardápio Digital;

3. simule uma jornada até a conclusão do pedido;

4. confirme se o evento `purchase` aparece no fluxo;

5. verifique se o acionador foi ativado;

6. confirme se a tag do GA4 disparou corretamente.

Esse passo evita publicar configurações incompletas ou com parâmetros errados.

<Image src="https://blob-cdn.documentation.ai/org-daf9910e-bdcf-4cb1-ab83-4cd8623e3f5e/doc-1c1fae12-7678-4e2e-90b6-bf66ffac55be/1773962295206-j5pjvpvzm4g-test-event-purchase.test-event-purchase" alt="" caption="Modo Preview do Google Tag Manager exibindo o evento purchase no fluxo de depuração, com a tag GA4 - purchase disparada corretamente após a conclusão do pedido no Cardápio Digital." style="width: 100%; height: 100%; border-radius: 12px;" />

> **Importante:** mesmo com o evento disparando corretamente no modo Preview do Google Tag Manager, os dados podem levar de **24 a 48 horas** para aparecer de forma completa nos relatórios do Google Analytics 4. Para validação imediata, utilize o **Realtime** e, quando necessário, o **DebugView**.
